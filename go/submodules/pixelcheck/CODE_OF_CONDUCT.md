# Code of Conduct

PixelCheck adopts the [Contributor Covenant 2.1] as the canonical
statement of community standards in this project. We incorporate the
document by reference rather than copying its text, so that upstream
wording (clarifications, tone improvements, future minor updates) applies
automatically and there is no risk of drift between repositories in the
`@xcodethink` org.

[Contributor Covenant 2.1]: https://www.contributor-covenant.org/version/2/1/code_of_conduct/

This convention matches the Linux Foundation, Node.js, Kubernetes,
Apache, and most GitHub-native open-source projects.

## Scope

The community standards apply across:

- This GitHub repository — issues, pull requests, discussions, commits, wikis, releases
- Maintainer-provided communication channels (email, chat, video calls)
- Project-affiliated public events (talks, demos, hackathons)
- Any chat / forum channel established under the project name

## Reporting

If you experience or witness conduct that does not meet the community
standards documented in the canonical link above:

1. **Preferred** — open a confidential report via GitHub Security Advisory:
   <https://github.com/xcodethink/pixelcheck/security/advisories/new>

   While GHSA is primarily a vulnerability disclosure channel, GitHub
   permits sensitive personal-impact disclosures through the same flow
   because it is encrypted, auditable, and shields the reporter from
   public visibility while the report is investigated.

2. **Alternative** — contact the maintainer team listed at
   <https://github.com/xcodethink>

A dedicated CoC email channel may be added in a future minor release as
the project grows. Until then, the GHSA flow is the recommended route.

## Response and enforcement

The maintainer team reviews every report and responds within 7 calendar
days. Enforcement decisions follow the [Enforcement Guidelines]
published alongside the Contributor Covenant — the four-step ladder
from private correction to permanent removal — applied consistently
regardless of the contributor's affiliation, sponsorship, or prior
contribution volume.

[Enforcement Guidelines]: https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines

Reports are kept confidential. Reporters and witnesses will not face
adverse action by the maintainer team for reporting in good faith.

## Escalation

If you believe a report has been mishandled or maintainers are
unresponsive, you can escalate to GitHub's Trust & Safety team through
the **Report abuse** flow on the relevant issue, pull request, or user
profile, or via <https://support.github.com/>.
