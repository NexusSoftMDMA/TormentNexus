# Release v1.1.1 — Patch: dynamic version reading

- **Version**: 1.1.1
- **Date**: 2026-05-03
- **Previous**: [v1.1.0](2026-05-03-v1.1.0.md) (~30 minutes earlier)
- **Tarball**: 624.1 kB / 368 files
- **Tag**: `v1.1.1`

## Summary

**Fixes a v1.1.0 regression**: v1.1.0 shipped with `"1.0.1"` strings still hardcoded in 4 source locations, so `pixelcheck --version`, SARIF reports, MCP server self-identification, and `list_capabilities` all reported the wrong version.

Recommended for all v1.1.0 users.

## Code changes

### Fixed
- `src/cli.ts:152` — `pixelcheck --version`
- `src/core/ci-reporters.ts:223` — SARIF `tool.driver.version` field (real protocol-layer info, GitHub Code Scanning may filter on it)
- `src/mcp/server.ts:104` — MCP server self-identification (AI clients may log this)
- `src/mcp/tools/list-capabilities.ts:45` — `list_capabilities` response

### Added
- `src/core/version.ts` — `getPackageVersion()` reads `package.json#version` at runtime from a fixed path (two levels up from itself). Single source of truth for the package version. Future releases only need to bump `package.json#version` — no source-string chasing.

### Test fixture
- `docs/integration/fixture-sarif.json` updated to `1.1.1` (the SARIF byte-identical test asserts equality with this fixture; the version field is now part of the dynamic-read result).
