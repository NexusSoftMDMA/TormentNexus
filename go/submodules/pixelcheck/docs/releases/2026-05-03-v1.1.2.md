# Release v1.1.2 — Repository hygiene + cross-project reference scrub

- **Version**: 1.1.2
- **Date**: 2026-05-03
- **Previous**: [v1.1.1](2026-05-03-v1.1.1.md)
- **Tarball**: ~624 kB / 368 files (unchanged from v1.1.1)
- **Tag**: `v1.1.2`

## Summary

Internal cleanup release. No public API changes. No new features. Functionally identical to v1.1.1 for end users. The repository is cleaner: internal planning files and cross-project references have been removed, and the critic prompt's example brand names are now generic.

Recommended upgrade for v1.1.x users for forward consistency, but no urgency.

## Code changes

### `src/core/critic.ts`
The `localization` rule in the critic prompt template named two specific brand examples (third-party project names) as illustrations of "exempt brand names." Replaced with a generic phrasing — "the audited site's brand and trademarks" — that doesn't tie the package to any third-party project.

Functional behaviour: **identical**. The exemption logic operates on the audited site's actual brand names at runtime, regardless of which examples appear in the prompt template.

## Repository hygiene (no impact on shipped tarball)

The following files were removed from the working tree AND from git history via `git filter-repo`. None of them were in any published npm tarball (verified by `npm pack --dry-run` against v1.0.0–v1.1.1).

- `PLAN.md` (root)
- `docs/research/` (directory)
- `docs/launch-post.md`, `docs/launch-post-zh.md`, `docs/show-hn.md`
- `docs/SLO.md`
- `docs/release-notes/` (legacy duplicate of `docs/releases/`)
- `docs/archive/` (legacy launch material)
- `docs/assets/og-image.png` + `.svg` (OG images belong in GitHub Settings → Social preview, not in the repo per industry-standard OSS practice)
- One `docs/releases/` entry that was an internal rehearsal record rather than a published release

The removal aligns the repository with the standard set by major reference OSS libraries (React, Vue, Vitest, Playwright, Hono, Zod, Drizzle ORM, Stagehand) — none of which carry marketing material, internal planning, SLO commitments for non-services, or social-preview assets in their source trees.

## Documentation sanitization

5 ADRs (ADR-001 / ADR-002 / ADR-004 / ADR-032 / ADR-033) had references to project-internal planning paths or sibling-repo filesystem paths. These have been replaced with generic descriptions so the ADRs document the technical decision context only.

`docs/ci-integration.md` and `docs/writing-scenarios.md` had third-party brand examples (a sibling project's name); replaced with generic placeholders.

`docs/integration/fixture-sarif.json` updated to 1.1.2 to match the dynamic-read tool driver version.
