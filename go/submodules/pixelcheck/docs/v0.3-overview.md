# v0.3 Overview — What changed since v0.2

v0.3 is a major upgrade focused on **reliability**, **cost**, **observability**,
and **ecosystem integration**. The tool is still local-first, single-binary,
zero-server, but it now has the primitives a commercial product demands:
measured reliability (benchmark harness), observable runs (live dashboard +
explorer), agent learning (memory + plan cache), cost tiers, and first-class
entry points (MCP, Chrome extension, persona generator).

## Tour of what's new

### 1. Signal-based convergence (Week 1)
The autonomous agent no longer trusts "click reported success". Each action is
scored against four additional dimensions:
- **network** — did the relevant API call return 2xx?
- **performance** — Core Web Vitals within budget?
- **error** — no new console / page errors?
- **interaction** — did anything on the page actually change?

Writers add `verification: network|performance|error|interaction` in their
scenario's `success_criteria`. See [acceptance-criteria.md](acceptance-criteria.md)
for patterns.

### 2. Cost tiers (Week 1)
`cost_mode: max | balanced | economy` in project config. Balanced (default) uses
Haiku navigator with Sonnet escalation on low confidence — **~3-5× cheaper** per
action at comparable success rate. Override per-run with `AUDIT_COST_MODE=economy`.

### 3. Plan cache + micro-replan (Week 1)
- Plan cache at `~/.ai-browser-auditor/plan-cache.db` — skips Opus planning on
  repeat runs against the same (site, persona class, DOM skeleton) combination.
  60-80% hit rate in practice.
- Micro-replan — on a stuck step, Haiku first tries to rewrite / skip / escalate
  before triggering a full Sonnet replan. ~15× cheaper when it works.

### 4. WebArena benchmark harness (Week 2)
`ai-audit benchmark --tasks <dir>` runs WebArena-compatible task JSON through
the autonomous agent. Reports pass@1, per-difficulty breakdown, p50/p95
duration, total cost. Drop in the official WebArena task set as-is.
Ships with `benchmarks/local-mini/` (3 tasks) running against the fixture site.

### 5. Critic calibration gate (Week 2)
`ai-audit calibrate` runs the vision critic against 5 labeled fixture
screenshots and gates CI on three thresholds: mean_agreement ≥ 0.85,
mean_max_distance ≤ 1.5, fully_aligned_rate ≥ 0.70. Detects model drift
when Anthropic ships a new Sonnet or when critic prompts change.

### 6. Observer timeline + multi-session grid (Week 3)
- Live dashboard now has a **timeline scrubber** — every action/plan/criterion
  as a clickable step; detail drawer shows meta + related events.
- `/grid` route renders an N-up grid when a run executes multiple (persona ×
  scenario) units in parallel. Status badge, cost, step count, last step
  label per tile, 2-second polling.

### 7. Report explorer SPA (Week 3)
Every run now also writes `audit-explorer.html` — a filterable single-file
explorer. Filter by persona × scenario × status × dimension-score-ceiling ×
issue severity. Gantt-style timing bars per step. XSS-hardened JSON embed.

### 8. MCP server (Week 4)
`ai-audit-mcp` — stdio MCP server exposing 6 tools (audit_url, explore_url,
list_personas, list_scenarios, calibrate_critic, get_last_report). Drop
into `~/.mcp.json` to let any MCP-aware agent (Claude Code, Cursor, Cline,
Continue, Zed) run audits inline.

### 9. Agent memory (Week 4)
`~/.ai-browser-auditor/memory.db` stores per-(host, persona-class) facts
learned across runs. Facts feed the planner prompt as hints on first plan —
cutting replans and wrong clicks on repeat visits. Confidence-weighted,
TTL'd, contradictable.

### 10. Persona data pipeline (Week 5)
`ai-audit persona generate --country=BR --device=mobile` — deterministic
persona YAML from curated market-data (StatCounter + Cloudflare Radar +
Ookla Q1 2026). 17 countries covered. Hand-written personas still work;
this is additive.

### 11. Scenario recorder Chrome extension (Week 5)
`extensions/scenario-recorder/` — MV3 extension, click-through →
scenario YAML export. Privacy-hardened (no passwords, no network).
Drops scenario-authoring time from minutes to seconds.

## Migration from v0.2

**Nothing breaks.** All new fields are optional or defaulted; all new
commands are additive. Existing v0.2 scenarios / personas / projects
work unchanged. Recommended migration steps:

1. `npm install` (pulls in `@modelcontextprotocol/sdk`, `ws`)
2. Run existing scenarios once — they'll pick up the cost_mode='balanced'
   default and the new signals on any autonomous scenarios, but behave
   identically for scripted ones.
3. Optionally regenerate hand-written personas with
   `ai-audit persona generate --country=… --out personas/` to pick up the
   market-data-driven mental_model notes.
4. Optionally register `ai-audit-mcp` in `~/.mcp.json` for inline agent use.
5. Optionally add `ai-audit calibrate` to CI for drift detection.
