## Description

GitHub #120: Alan has `admin` + locked-down `gateway` Hermes profiles; ScarfGo lists them (read-only) but can't switch/operate against them. Add a profile switcher.

**Chosen design: B — per-connection profile scoping (NOT global `hermes profile use`).** ScarfGo points its own reads/writes/CLI at `<base>/profiles/<name>` via `HERMES_HOME` / `remoteHome`, WITHOUT mutating the host's `~/.hermes/active_profile`. The Mac app, terminal, cron daemon, and running gateway on the host are untouched. This is the intended Hermes mechanism and avoids the destructive global side-effect that originally kept switching off-phone (ProfilesView.swift v2.6 note).

**Why not A (mirror Mac global switch):** Mac's canonical switch RELAUNCHES the whole app to flush in-process state (ProfilesViewModel.switchAndRelaunch) — iOS can't relaunch cleanly; and A mutates host-global state affecting Mac/terminal/gateway.

**Source-verified against installed Hermes 0.16 (~/.hermes/hermes-agent):**
- `hermes_constants.get_hermes_home()` reads `HERMES_HOME` env FIRST (line 73); it wins over active_profile. Docs: "subprocess spawners are expected to propagate HERMES_HOME explicitly."
- `hermes_cli/main.py:_apply_profile_override()` — if `HERMES_HOME` already points at a `profiles/<name>` dir, returns early and never reads active_profile (clobber-proof for named profiles). Root paths fall through to active_profile (the default-case edge).
- `-p/--profile <name>` flag (incl. `-p default`) forces a profile explicitly regardless of active_profile → robust lever for the chat/CLI path.

**Key existing seams reused:**
- `IOSServerConfig.remoteHome` → `ServerContext.paths.home` → every HermesPathSet path (state.db, memories, cron, sessions, gateway_state, scarf/). Set remoteHome to the profile dir → ALL direct-file/DB surfaces follow for free.
- `HermesProfileResolver` is LOCAL-ONLY; remote home is static `~/.hermes` today (the latent half-switch bug this fixes).
- ACP chat command built in ACPClient+iOS.swift; transport command builders in CitadelServerTransport.

**Phases (each: plan→execute→test→fresh-eyes audit→commit):**
- B0 resolution helper + per-server selection persistence (pure logic + storage)
- B1 wire selection into ServerContext; rebuild tab subtree on change (direct reads follow)
- B2 profile-scope chat ACP + scoped CLI via `-p <name>` (default-safe)
- B3 UI switcher in ProfilesView (select/confirm/indicator/copy; restart live chat)
- B4 docs/wiki/memory sync + fresh-eyes integration audit

**Out of scope:** profile create/rename/delete/import/export (stay Mac-only); global `hermes profile use` from phone.

Branch: feature branch off main; never push until Alan says so.
