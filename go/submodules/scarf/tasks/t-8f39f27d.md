## Plan

## Plan
1. Add a small pure helper (ScarfCore or ScarfIOS) that resolves an effective Hermes home for `(baseHome, selectedProfile)`:
   - `default`/nil → baseHome unchanged (`remoteHome ?? "~/.hermes"`)
   - named `<n>` → `"<baseHome>/profiles/<n>"`
   - validate name against Hermes regex `^[a-z0-9][a-z0-9_-]{0,63}$`; invalid → treat as default (fail safe)
   - works for custom baseHome (Docker `/opt/data` → `/opt/data/profiles/<n>`)
2. Persist selected profile PER serverID (sidecar UserDefaults store, e.g. key `scarfgo.selectedProfile.<serverID>`) — avoid mutating IOSServerConfig/its persisted store (no migration). Default = nil/"default".
3. Provide get/set + a tiny observable or plain store accessor for later phases.

## Tests
- Resolution: default→base; named→base/profiles/name; custom base; invalid name→base; tilde base preserved.
- Persistence: set/get round-trip per serverID; isolation between two serverIDs; absent → default.
